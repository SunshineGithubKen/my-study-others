クラスとオブジェクト: 
クラスとオブジェクトを使うことで、データ型を定義し、オブジェクト指向プログラミングのメリットを享受できます。

アクセス修飾子: 
publicやprivateなどのアクセス修飾子を使用して、メンバ変数やメソッドへのアクセス範囲を制御できます。

継承: 
クラスの継承を使用して、既存のクラスを基にして新しいクラスを作成できます。これにより、コードの再利用性と保守性が向上します。

インターフェース: 
インターフェースを実装することで、クラスが特定のメソッドを持つことが保証されます。これにより、多重継承の代替手段としての柔軟性を提供します。

Classes and Objects:
By using classes and objects, you can define new data types and enjoy the benefits of object-oriented programming. A class serves as a blueprint, and an object is an instance created based on that blueprint. A class has attributes (member variables) and behaviors (methods), while an object holds the values of its attributes and the results of its methods.

Access Modifiers:
Access modifiers control the scope of access to class member variables and methods. The main access modifiers are public, private, and protected.

public: Accessible from anywhere.
private: Accessible only within the same class.
protected: Accessible within the same class and its subclasses.
Inheritance:
By using class inheritance, you can create a new class (child class or subclass) based on an existing class (parent class or superclass). The child class inherits the attributes and methods of the parent class and can add new attributes and methods. This enhances code reuse and improves maintainability.

Interface:
By implementing an interface, a class ensures that it has specific methods. An interface can contain abstract methods only, and the actual implementation is done by the class that implements the interface. This provides flexibility as an alternative to multiple inheritance, even in languages that do not support it directly.
